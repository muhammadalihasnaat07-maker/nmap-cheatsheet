Network Mapper — Complete Command & Script Reference

Scan Types
TCP Scan Techniques
CommandDescription
nmap -sS <target>
SYN (Stealth) Scan — Sends SYN, never completes handshake. Fast and less detectable by firewalls/logs. Requires root.
nmap -sT <target>
TCP Connect Scan — Full 3-way handshake. No root needed, but noisy and logged by target OS.
nmap -sA <target>
ACK Scan — Sends ACK packets to map firewall rules and detect filtered vs unfiltered ports.
nmap -sW <target>
Window Scan — Like ACK scan but examines TCP window field. Can differentiate open/closed on some systems.
nmap -sM <target>
Maimon Scan — Sends FIN/ACK probe. Works on some BSD systems where closed ports drop the packet.
nmap -sN <target>
TCP Null Scan — No flags set. Closed ports respond with RST; open/filtered ports don't respond.
nmap -sF <target>
FIN Scan — Only FIN flag set. Bypasses some non-stateful firewalls. RFC-compliant behavior expected.
nmap -sX <target>
Xmas Scan — Sets FIN, PSH, URG flags. Named for the "lit up" packet. Evades some packet filters.
Other Scan Protocols
CommandDescription
nmap -sU <target>
UDP Scan — Scans UDP ports. Slower than TCP; ICMP "port unreachable" = closed, no response = open|filtered.
nmap -sY <target>
SCTP INIT Scan — Like TCP SYN but for SCTP. Detects VoIP/telecom services. Requires root privileges.
nmap -sZ <target>
SCTP COOKIE-ECHO — Sends COOKIE-ECHO chunk; open ports don't respond, closed send ABORT.
nmap -sO <target>
IP Protocol Scan — Discovers which IP protocols (TCP, UDP, ICMP, IGMP…) are supported by target host.
nmap -b <FTP> <target>
FTP Bounce Scan — Uses FTP server as proxy to scan another host. Mostly patched on modern FTP servers.
nmap -sn <target>
Ping Scan (No Port Scan) — Host discovery only. Uses ICMP echo, TCP SYN to port 443, TCP ACK to port 80.
nmap -Pn <target>
Skip Host Discovery — Treats all hosts as online. Use when ICMP is blocked and host appears "down."
nmap -PR <target>
ARP Ping Scan — Uses ARP for LAN host discovery. Faster and more reliable on local networks.
Target Specification & Port Options
Target Specification
CommandDescription
nmap 192.168.1.1
Single IP — Scan one specific host by IPv4 address.
nmap 192.168.1.1-254
IP Range — Scan all IPs in the range 192.168.1.1 to 192.168.1.254.
nmap 192.168.1.0/24
CIDR Notation — Scan entire /24 subnet (256 addresses). Supports /8, /16, /32 etc.
nmap -iL targets.txt
Input from File — Read targets from a text file, one per line. Supports IPs, ranges, hostnames.
nmap --exclude <IP>
Exclude Host — Skip specific IPs from the scan. Use --excludefile file.txt for multiple.
nmap -6 <target>
IPv6 Scan — Enable IPv6 scanning. Target must be specified as IPv6 address or hostname.
nmap scanme.nmap.org
Hostname Scan — Resolves hostname via DNS then scans. DNS resolution can be disabled with -n.
Port Specification
CommandDescription
nmap -p 80 <target>
Single Port — Scan only port 80 on the target host.
nmap -p 80,443,22
Multiple Ports — Comma-separated list of specific ports to scan.
nmap -p 1-1024
Port Range — Scan ports 1 through 1024 (well-known ports).
nmap -p-
All 65535 Ports — Scan every TCP port. Very thorough, but slow. Alias for -p 1-65535.
nmap -p U:53,T:80
Protocol-specific Ports — Specify UDP (U:) or TCP (T:) ports explicitly in one command.
nmap --top-ports 100
Top N Ports — Scan the N most commonly open ports based on frequency data.
nmap -F <target>
Fast Mode — Scans only the 100 most common ports. Much faster than default 1000-port scan.
nmap -r <target>
Sequential Scan — Don't randomize port order. Useful for reproducible results or rate-limited testing.
Service, Version & OS Detection
Detection Options
CommandDescription
nmap -sV <target>
Service Version Detection — Probes open ports to determine service name and version (e.g., Apache 2.4.51).
nmap -sV --version-intensity 9
Max Intensity Version Scan — 0–9 scale. Higher = more probes sent = more accurate but slower detection.
nmap -sV --version-light
Light Version Scan — Intensity 2. Fastest version detection with lower accuracy. Good for quick assessments.
nmap -sV --version-all
All Version Probes — Tries every probe. Slowest but most thorough — intensity 9 equivalent.
nmap -O <target>
OS Detection — Uses TCP/IP stack fingerprinting to identify operating system and version. Requires root.
nmap -O --osscan-guess
Aggressive OS Guess — Guesses OS when no perfect match found. Shows percentage confidence.
nmap -O --osscan-limit
Limit OS Scan — Only performs OS detection against hosts with at least one open and one closed port.
nmap -A <target>
Aggressive Scan — Enables -sV, -O, script scanning (-sC), and traceroute in one flag.
Timing & Performance
CommandDescription
nmap -T0 <target>
Paranoid — slowest One probe every 5 minutes. Max IDS evasion. For extremely sensitive environments.
nmap -T1 <target>
Sneaky — One probe every 15 seconds. Slow enough to evade most IDS/IPS systems.
nmap -T2 <target>
Polite — Slower scan to reduce bandwidth and target load. Less likely to crash services.
nmap -T3 <target>
Normal — default Balanced speed. Nmap's standard timing when no -T flag is given.
nmap -T4 <target>
Aggressive — Assumes fast network. Speeds up scan significantly. Common choice for internal networks.
nmap -T5 <target>
Insane — fastest Extremely fast. May miss results on slow/unreliable networks. High packet loss risk.
nmap --min-rate 500
Min Packet Rate — Never send fewer than N packets per second. Ensures minimum throughput.
nmap --max-retries 2
Limit Retries — Cap probe retransmissions per port. Reduces total scan time significantly.
Firewall / IDS Evasion & Spoofing
CommandDescription
nmap -f <target>
Fragment Packets — Splits TCP headers across multiple tiny IP packets to confuse packet inspection and IDS systems.
nmap --mtu 16 <target>
Custom MTU — Set your own fragmentation offset size (must be multiple of 8). Bypasses some deep packet inspection.
nmap -D RND:10 <target>
Decoy Scan — Spoofs 10 random decoy IPs alongside your real IP. Obscures the actual scanning host in logs.
nmap -S 1.2.3.4 <target>
Spoof Source IP — root only Forges the source IP. Responses go to the spoofed address; you won't see replies.
nmap -e eth0 <target>
Specify Interface — Force Nmap to use a specific network interface for sending/receiving packets.
nmap --source-port 53
Spoof Source Port — Set source port to 53 (DNS). Some firewalls whitelist DNS traffic, allowing bypass.
nmap --data-length 25
Append Random Data — Adds 25 bytes of random data to packets, changing their size to confuse signature matching.
nmap --randomize-hosts
Randomize Host Order — Scans targets in random order to avoid pattern detection on monitored networks.
nmap --spoof-mac Apple
Spoof MAC Address — Uses a MAC from a specific vendor (Apple, Cisco, etc.) or "0" for random MAC. LAN only.
nmap --proxies http://proxy:8080
HTTP/SOCKS4 Proxy — Route TCP connections through a proxy chain. Hides scanner's true IP address.

Verbosity & Misc Options
CommandDescription
nmap -v <target>
Verbose Mode — Prints more info during scan. Use -vv or -v2 for extra verbosity.
nmap -d <target>
Debug Mode — Level 1–9 debugging output. -d2 for deeper. Very verbose logs for troubleshooting.
nmap --open <target>
Show Only Open Ports — Filters output to only display ports with "open" state. Cleaner results.
nmap --reason <target>
Show Reason — Explains why each port is in its state (e.g., "syn-ack", "reset", "no-response").
nmap --resume scan.gnmap
Resume Scan — Continues a previously interrupted scan from a grepable output file.
NSE — Nmap Scripting Engine
Script Execution Options
CommandDescription
nmap -sC <target>
Default Scripts — Runs all scripts in the "default" category. Equivalent to --script=default.
nmap --script=vuln
Vuln Category — Run all vulnerability detection scripts. Checks for known CVEs and misconfigurations.
nmap --script=http-*
Wildcard Match — Run all scripts whose name starts with "http-". Useful for targeting specific services.
nmap --script "not intrusive"
Exclude Category — Run everything except "intrusive" scripts. Safer for production environments.
nmap --script-args n=val
Pass Script Arguments — Provide parameters to scripts. E.g., credentials, paths, timeouts.
nmap --script-updatedb
Update Script DB — Rebuilds the script database after adding custom NSE scripts to the scripts/ directory.
nmap --script-help http-title
Script Help — Displays documentation, arguments, and usage examples for a specific NSE script.

Key NSE Scripts Reference
HTTP / Web Scripts
Script NameCategoryPurpose & Use Case
http-title
default
Retrieves the HTML page title from HTTP services. Quickly identifies what web app is running on open port 80/443.
http-headers
safe
Fetches HTTP response headers. Reveals server type, cookies, security headers (X-Frame-Options, HSTS, CSP), and tech stack.
http-methods
safe
Enumerates allowed HTTP methods (GET, POST, PUT, DELETE, TRACE…). TRACE and PUT enabled can indicate security misconfigurations.
http-robots.txt
default
Reads the robots.txt file from the web server. Often exposes hidden or sensitive directory paths that admins want to hide.
http-auth
discovery
Checks if a web server requires HTTP authentication and reports the authentication scheme (Basic, Digest, NTLM).
http-brute
brute
Performs brute-force password attacks against HTTP Basic/Digest/Form authentication endpoints using wordlists.
http-enum
discovery
Web directory enumeration — Tries common paths (/admin, /login, /phpmyadmin…) to find hidden resources and installed web apps.
http-shellshock
exploit
Checks for the Shellshock (CVE-2014-6271) vulnerability in CGI scripts. A critical bash injection flaw in web servers.
http-sql-injection
intrusive
Spider-based SQL injection detection — crawls the site and tests GET/POST parameters for SQL injection vulnerabilities.
http-wordpress-enum
discovery
Enumerates WordPress plugins, themes, and users. Critical for assessing WordPress site attack surface.
http-vuln-cve2017-5638
vuln
Detects Apache Struts2 S2-045 RCE (Equifax breach CVE). Sends crafted Content-Type header to trigger remote code execution.
http-cookie-flags
safe
Inspects cookie security flags — checks if cookies are missing HttpOnly, Secure, or SameSite attributes, which can lead to session hijacking.
SSL / TLS & Certificate Scripts
Script NameCategoryPurpose & Use Case
ssl-enum-ciphers
safe
Lists all supported SSL/TLS cipher suites and grades them (A–F). Identifies weak ciphers like RC4, DES, EXPORT-grade encryption.
ssl-cert
default
Retrieves and displays SSL/TLS certificate details — subject, issuer, validity dates, SANs. Useful for cert auditing and finding expired certs.
ssl-heartbleed
vuln
Tests for Heartbleed (CVE-2014-0160) — the critical OpenSSL memory disclosure flaw that leaked private keys and session data.
ssl-poodle
vuln
Checks for POODLE vulnerability (CVE-2014-3566) — tests if SSL 3.0 is enabled, allowing CBC padding oracle attacks on HTTPS.
ssl-dh-params
safe
Detects weak Diffie-Hellman parameters (Logjam attack). Identifies use of common DH primes vulnerable to pre-computation attacks.
tls-alpn
safe
Enumerates Application-Layer Protocol Negotiation (ALPN) protocols — identifies if HTTP/2, SPDY, or other protocols are available.
Network Services Scripts (SMB / FTP / SSH / DNS)
Script NameCategoryPurpose & Use Case
smb-vuln-ms17-010
vuln
Detects EternalBlue (WannaCry CVE-2017-0144) — the most critical SMB vulnerability. A hit means the system is likely fully compromisable.
smb-vuln-ms08-067
vuln
Tests for MS08-067 (Conficker worm) — A pre-auth RCE in Windows Server Service. Affects unpatched Windows XP/2003/Vista/2008.
smb-enum-shares
discovery
Enumerates all SMB shares on target including hidden shares (C$, ADMIN$, IPC$). Shows share names, type, and comment fields.
smb-enum-users
discovery
Retrieves Windows user account list via SMB. Account names useful for targeted brute force or password spraying attacks.
smb-os-discovery
default
Identifies Windows OS version via SMB — more reliable than TCP/IP fingerprinting. Reveals computer name, domain, and workgroup info.
smb-security-mode
default
Reports SMB authentication and security mode — whether SMB signing is required, guest logins allowed, and challenge/response type used.
smb2-security-mode
default
Checks SMBv2/v3 security configuration — message signing required/enabled. Disabled signing allows man-in-the-middle relay attacks.
ftp-anon
default
Checks if FTP anonymous login is permitted. Anonymous access to FTP is a critical misconfiguration exposing file system access.
ftp-brute
brute
Performs FTP brute-force using built-in username/password lists. Requires --script-args ftp-brute.userdb=users.txt for custom lists.
ftp-bounce
intrusive
Tests if an FTP bounce attack is possible using the PORT command — allows using FTP server as a proxy to scan third-party hosts.
ssh-auth-methods
safe
Enumerates allowed SSH authentication methods (password, publickey, keyboard-interactive). Password auth alone is a risk indicator.
ssh-brute
brute
Brute-forces SSH login credentials. Supports username/password lists. Combine with --script-args userdb=,passdb=.
ssh-hostkey
default
Retrieves and prints SSH server host keys (RSA, DSA, ECDSA, Ed25519). Used to verify server identity and detect MITM attacks.
dns-brute
discovery
DNS subdomain brute-force — tries common subdomain names against the target domain. Discovers hidden subdomains like mail., vpn., admin.
dns-zone-transfer
intrusive
Attempts a DNS zone transfer (AXFR) to get full DNS record dump. A misconfigured DNS server will return all records for the domain.
dns-recursion
safe
Checks if a DNS server has open recursion enabled — open resolvers can be abused for DNS amplification DDoS attacks.
Database, Mail & Infrastructure Scripts
Script NameCategoryPurpose & Use Case
mysql-empty-password
intrusive
Checks if MySQL root account has no password — one of the most common database misconfigurations leading to full data exposure.
mysql-databases
discovery
Lists all MySQL databases on the server. Requires valid credentials via --script-args mysqluser=root,mysqlpass=.
mysql-brute
brute
Brute-forces MySQL credentials using username/password lists. Identifies weak or default database credentials.
mssql-info
default
Gathers Microsoft SQL Server information — version, edition, named instance, SQL Server Browser service details on port 1433.
mssql-brute
brute
Brute-forces MSSQL SA and other accounts. SA (System Administrator) with weak passwords gives full database server control.
redis-info
safe
Retrieves Redis server information — version, memory usage, connected clients. Unauthenticated Redis can expose full in-memory database.
mongodb-info
default
Fingerprints MongoDB server — version, host info, databases list. MongoDB without auth enabled is a critical misconfiguration.
smtp-enum-users
intrusive
Enumerates email user accounts via SMTP using VRFY, EXPN, or RCPT TO commands. Identifies valid usernames for further attacks.
smtp-open-relay
safe
Tests if SMTP server is an open relay — can it forward email for arbitrary domains? Open relays are used for spam distribution.
snmp-info
default
Gathers SNMP system information — sysDescr, sysName, sysLocation. Reveals device info when default community strings (public/private) are in use.
snmp-brute
brute
Brute-forces SNMP community strings. Found strings grant read (public) or write (private) access to network device configurations.
ldap-search
safe
Performs LDAP queries to enumerate Active Directory objects, users, groups, and OUs. Requires valid bind credentials or anonymous access.
rdp-enum-encryption
safe
Enumerates RDP encryption settings — identifies if NLA is enforced or if weak encryption is permitted on Remote Desktop (port 3389).
rdp-vuln-ms12-020
dos
Tests for MS12-020 RDP DoS vulnerability — a pre-auth denial of service flaw in Remote Desktop Protocol. Can crash unpatched systems.
vnc-info
default
Retrieves VNC server banner and protocol version. VNC without authentication is a critical misconfiguration giving full remote desktop access.
vnc-brute
brute
Brute-forces VNC passwords. VNC uses single-password auth, making weak passwords especially dangerous for remote desktop access.
telnet-brute
brute
Brute-forces Telnet login credentials. Telnet sends data in plaintext — combined with weak passwords, it's a critical risk on IoT/legacy devices.
Common Recipes & Power Combos
Quick Recon
# Full aggressive scan
nmap -A -T4 <target>

# Quick ping sweep
nmap -sn 192.168.1.0/24

# Fast 100 ports
nmap -F -T4 <target>

# All ports + services
nmap -p- -sV <target>
Vulnerability Scan
# All vuln scripts
nmap --script vuln <target>

# EternalBlue check
nmap -p445 --script smb-vuln-ms17-010

# SSL cipher audit
nmap -p443 --script ssl-enum-ciphers

# Heartbleed test
nmap -p443 --script ssl-heartbleed
Stealth & Evasion
# Stealth with decoys
nmap -sS -D RND:5 <target>

# Slow IDS-evasion
nmap -T1 -f <target>

# DNS bypass source
nmap --source-port 53 <target>

# Spoof MAC (LAN)
nmap --spoof-mac 0 -sS <target>
⚠ Legal Reminder: Only scan systems you own or have explicit written permission to test. Unauthorized port scanning is illegal in many jurisdictions. Use scanme.nmap.org for authorized practice scanning. Always document your scope in a Rules of Engagement (RoE) before any penetration test.
